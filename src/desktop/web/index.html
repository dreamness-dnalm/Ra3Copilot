当前工作区

Provider

配置大模型供应商、Base URL 和 API Key。一个供应商可以保留多套不同的 Provider 配置。

已配置 Provider

模型信息会保存在本机配置中，重启后会直接显示。

可用模型

拉取 Provider 后，在这里选择要启用的模型。

Model

管理可用模型、默认模型、温度和输出长度。删除 Provider 会自动删除对应模型。

可用模型

可手动添加无法从 Provider 拉取列表的模型。

模型参数

模型 选择左侧列表中的模型后编辑。
Provider 新增模型时从已配置 Provider 中选择。
Model ID 填写供应商接口使用的真实模型名。
启用状态 关闭后不会出现在运行时可选模型中。
Temperature 越低越稳定，越高越发散。
Max tokens 留空或 0 表示使用供应商默认值。
Context length 用于展示和后续上下文预算；0 表示未知。
计费货币 默认人民币；仅影响后续新 run 的费用账本。
价格 / 百万 token 留空或 0 表示不计算费用；缓存输入只使用 Provider 返回的 cached tokens。
操作 修改后会重置下一轮 Agent 的模型缓存。
