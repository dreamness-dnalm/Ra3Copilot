可用模型

Model

管理可用模型、默认模型、温度和输出长度。删除 Provider 会自动删除对应模型。

可用模型

模型参数

模型 选择左侧列表中的模型后编辑。
Provider 新增模型时从已配置 Provider 中选择。
Model ID 填写供应商接口使用的真实模型名。
启用状态 关闭后不会出现在运行时可选模型中。
Temperature 越低越稳定，越高越发散。
Max tokens 留空或 0 表示使用供应商默认值。
Context length 用于展示和后续上下文预算；0 表示未知。
计费货币 默认人民币；仅影响后续新 run 的费用账本。
价格 / 百万 token 留空或 0 表示不计算费用；缓存输入只使用 Provider 返回的 cached tokens。
操作 修改后会重置下一轮 Agent 的模型缓存。

开启后会把后续 Agent run 发送到 LangSmith，便于查看链路、工具调用和模型耗时。

LangSmith

启用观测 默认关闭；保存后从下一轮对话开始生效。
LangSmith URL 例如 https://apac.api.smith.langchain.com
Token LangSmith API Key，仅保存在本机用户配置中。
Project LangSmith 项目名，用于聚合本软件的 run。
状态 观测默认关闭。

新建地图工程

工程名称 会作为工程列表显示名；保存目录可使用默认值，也可以手动指定。
保存目录 默认保存在用户数据的 projects 目录下；目录不存在会自动创建。
操作 新工程不能保存到非空文件夹；保存目录不能包含 Windows 非法字符。

新增 Provider

Provider 名称 名称必须唯一，用于区分不同 Provider 配置。

API Key 保存在本机用户数据目录，用于测试连接和运行模型。
操作 保存后可以测试连接并拉取模型。
